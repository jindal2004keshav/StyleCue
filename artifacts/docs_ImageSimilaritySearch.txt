API Contract — Image Similarity Search

Endpoint

- Method: POST
- Path: /brandeye/image_similarity_search
- Content-Type: multipart/form-data
- Auth: none enforced in code
- Host: https://ai-search-brandeye.blr.streamoid.com

Purpose

- Search products by vector similarity (image and/or text) plus optional metadata filters.
- Supports filter-only searches (no image, no text).
- Sorting by popularity or recency (see Sorting).

———

Query Parameters

- offset (int, optional, default 0)
    - Used only when vector search is active.
- limit (int, optional, default 100)
    - Max results; used in vector search and filter-only search.
- scroll_token (string, optional)
    - Only used for filter-only search pagination.
    - Treat as opaque; produced by the API as next_token.

———

Form Fields (multipart/form-data)

All list fields are currently parsed from the first value only, and split by comma. That means you should send a single comma-separated string per
field.

- departments (string, optional)
    - Comma-separated; maps to payload key Department.
    - Example: Women,Men
- categories (string, optional)
    - Comma-separated; maps to payload key category.
- colors (string, optional)
    - Comma-separated; maps to payload key Color.
- types (string, optional)
    - Comma-separated; maps to payload key Type.
- store_uuid (string, optional)
    - Comma-separated store UUIDs.
    - Unlike other fields, multiple values are merged if you send repeated form fields.
- brands (string, optional)
    - Comma-separated; maps to payload key brand.
- source (string, optional)
    - Comma-separated; maps to payload key source.
- trend_times (string, optional)
    - Comma-separated date range for payload key created_ts.
    - One value: 2025-01-01T00:00:00Z (interpreted as >=).
    - Two values: start,end (interpreted as >= start and <= end).
- discount_range (string, optional)
    - Comma-separated numeric range for payload key discount.
    - One value: 20 (interpreted as >= 20).
    - Two values: 10,50.
- selling_price_range (string, optional)
    - Comma-separated numeric range for payload key selling_price.
- query_text (string, optional)
    - Text used to generate text embeddings.
- image (file, optional)
    - Raw image used to generate image embeddings.
- sort_flag (string, optional, default popularity)
    - Implemented behavior: only popularity and recency are respected.
    - Any other value is coerced to recency.

Input Notes

- At least one of query_text, image, or any filter should be provided.
- If both image and query_text are provided, embeddings are blended with weights text=0.4, image=0.6.
- For list fields, pass a single comma-separated string. The current implementation only reads the first list value.

———

Sorting

- popularity → payload trending_score descending.
- recency → payload created_ts descending.
- relevance is mentioned in docstring but not honored by code. Any value other than popularity or recency becomes recency.

———

Pagination

Vector Search (image/text present)

- Uses offset and limit.
- scroll_token is ignored.
- Response next_token is always null.

Filter-Only Search

- Uses keyset pagination; offset is not used.
- Pass scroll_token from the previous response’s next_token.

———

Response Shape

Top-level wrapper:

{
  "image_similarity": {
    "results": [ ... ],
    "next_token": "..."
  }
}

Vector Search Results
Each item has:

- point (object): Qdrant payload (product metadata)
- point_id (string or int): Qdrant point id
- score (float): similarity score
- source (string): "image_embeddings_refined"

Filter-Only Results
Each item has:

- point (object): Qdrant payload
- point_id (string or int)
- source (string): "filters_only"
- No score field.

———

Status Codes

- 200 Success
- 422 Validation error (FastAPI)
- 500 Internal error
    - Response shape: { "detail": "Internal Server Error: <message>" }

Implementation Notes for Client Utility

- Always send multipart/form-data, even for text-only or filter-only queries.
- Send list fields as a single comma-separated string.
- Treat next_token as opaque and pass it back as scroll_token for filter-only pagination.
- Do not rely on total or scroll_token in response; the API returns next_token.

Form Fields (multipart/form-data)

All list fields are defined as list[str] in FastAPI, but the search logic reads only the first list value and then splits it by comma.
Therefore: send a single comma‑separated string per field (no spaces unless the payload value includes spaces intentionally).

- departments (string, optional)
  Comma‑separated list mapped to payload key Department.
  Example: Women,Men
- categories (string, optional)
  Comma‑separated list mapped to payload key category.
  Example: T-Shirts,Dresses
- colors (string, optional)
  Comma‑separated list mapped to payload key Color.
  Example: Black,Red
- types (string, optional)
  Comma‑separated list mapped to payload key Type.
  Example: Topwear,Bottomwear
- store_uuid (string, optional)
  Store UUIDs. Two supported input shapes:
    - Single field with commas: uuid1,uuid2 (will be split and trimmed)
    - Repeated fields: -F store_uuid=uuid1 -F store_uuid=uuid2 (will be used as-is)
- brands (string, optional)
  Comma‑separated list mapped to payload key brand.
  Example: Zara,H&M
- source (string, optional)
  Comma‑separated list mapped to payload key source.
  Example: shopify,magento
- trend_times (string, optional)
  Comma‑separated list for payload key created_ts.
    - One value: 2025-01-01T00:00:00Z → interpreted as >=
    - Two values: 2025-01-01T00:00:00Z,2025-02-01T00:00:00Z → interpreted as >= start and <= end
- discount_range (string, optional)
  Comma‑separated numeric range for payload key discount.
    - One value: 20 → >= 20
    - Two values: 10,50
- selling_price_range (string, optional)
  Comma‑separated numeric range for payload key selling_price.
    - One value: 20 → >= 20
    - Two values: 20,100
- query_text (string, optional)
  Text used to generate text embeddings.
- image (file, optional)
  Raw image used to generate image embeddings.
- sort_flag (string, optional, default popularity)
  Only popularity and recency are honored; any other value is coerced to recency.

Input Notes (corrected to code behavior)

- Send multipart/form-data for all requests (even filter‑only).
- For departments, categories, colors, types, brands, source, trend_times, discount_range, selling_price_range:
  send a single comma‑separated string in one field.
  The server only reads the first value in the list and splits it on commas.
- Avoid spaces after commas unless the payload values themselves include spaces (the server does not trim values for most fields).
- store_uuid is the only field that supports repeated form keys and also supports comma‑separated input.
- If both image and query_text are present, embeddings are blended with weights text=0.4, image=0.6.
